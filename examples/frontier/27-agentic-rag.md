# Agentic RAG（把檢索也拉進迴圈：Self-RAG / CRAG）

- **族別**：五之補（search / 動態策略族）
- **控制骨架**：`agent 在迴圈裡自己決定「要不要查、查什麼、查夠了沒」`
- **重點**：檢索不再是一次性前處理，而是 agent 的動態決策。

## Prompt 範例（CRAG 風格：檢索 → 評估 → 不夠好就改寫/換源）

```text
問題：{{QUESTION}}

迴圈：
1. [DECIDE] 這題需要外部資料嗎？(Self-RAG) 若你的內部知識足夠且可靠 → 直接答並標 [NO-RETRIEVAL]。
   否則產一個檢索 query。
2. [RETRIEVE] 我回填檢索到的文件片段。
3. [GRADE] 逐一評估這些片段「是否真的能回答問題」：
   - 足夠 → 用它們作答，並標出每個關鍵論點來自哪一段。
   - 不足/離題 → [CORRECT]：改寫 query / 換關鍵字 / 要求換來源，回到步驟 2（最多 3 次）。
4. 作答時：只用檢索到的內容，缺依據的地方明說「資料不足」，不要腦補。
```

## 為什麼這樣做可以

- **一次性檢索常常抓錯或抓不夠**：傳統 RAG 把檢索當固定前處理，抓到爛文件也照用。Agentic RAG 讓模型**評估檢索品質**，不夠好就改寫 query 或換來源——把「檢索」變成有回饋的迴圈而非單向管線。
- **Self-RAG 的「要不要查」判斷**省掉無謂檢索（簡單題直接答），也避免「明明不知道卻硬掰」。
- **要求標出論點來源、缺依據就明說**是接地護欄，壓低幻覺——這正是 self-refine 需要的「外部訊號」，讓修正基於真實檢索結果而非感覺。

## 什麼時候用 / 不要用

- ✅ 需要外部/最新知識、且檢索品質不穩的問答與研究任務。
- ✅ 想讓模型自己判斷何時該查、查夠了沒。
- ❌ 純推理、不需外部知識——檢索是多餘開銷。

## 進階

- **MCTS-RAG**：把 [tree search](../21-tree-search.md) 套在「檢索 + 推理」交錯的決策上，用於更難的多跳問答。
