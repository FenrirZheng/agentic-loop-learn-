# Tree search / backtracking（展開多分支、評估、走不通就回溯）

- **族別**：五、Search / 動態策略
- **控制骨架**：`展開多個候選分支 → 對 frontier 節點打分 → 走最有希望的 → 死路就回溯`
- **重點**：LLM 版的 MCTS——不是一條路走到黑，而是可回頭。

## Prompt 範例

分步展開 + 評分 + 回溯（樹狀探索）：

```text
你在用樹狀搜尋解這題：{{PROBLEM}}

每一步這樣做：
1. [EXPAND] 從目前狀態，提出 3 個「不同的下一步走法」，標記為分支 a/b/c。
2. [SCORE] 對每個分支給 0–10 的「前景分數」（估計它通往正解的可能），並說明理由。
3. [SELECT] 走分數最高的分支，更新狀態。
4. [CHECK] 若目前路徑陷入死路 / 明顯走不通 / 分數全部很低，
   [BACKTRACK] 回到上一個有未探索分支的節點，改走次高分的分支。

維護一份 [FRONTIER]（還沒展開的候選節點 + 它們的分數），這樣回溯時知道往哪回。
找到解就輸出完整路徑；探索超過 {{N}} 步仍無解則回報目前最佳部分解。
```

## 為什麼這樣做可以

- **一條鏈走到黑會卡在錯誤前提上**：線性 CoT 一旦早期走錯，後面全錯且無法回頭。樹搜尋保留多個分支和 frontier，允許「這條死了就退回去換一條」，大幅提高找到正解的機會。
- **對 frontier 打分（value function）** 引導算力往有希望的方向花，而不是盲目窮舉所有分支——這是「算力花在哪比花多少重要」的體現。
- **顯式 backtrack + frontier 記錄**讓回溯有據可依，不會退回後又鬼打牆走回同一條死路。

## 什麼時候用 / 不要用

- ✅ 解空間有結構、可分步展開、且「早期選擇會鎖死後續」的任務：規劃、證明、遊戲、複雜程式合成。
- ✅ 願意用較高算力換更高成功率。
- ❌ 簡單、單路徑就能解的任務——樹搜尋是殺雞用牛刀，成本高。

## 進階：LATS（完整 agent 版）

**LATS（Language Agent Tree Search）** 把 ReAct + Tree-of-Thoughts + 反思 + MCTS 合一：用 value function 對 frontier 打分、可 backtrack、並把**失敗經驗寫成 reflection** 再繼續探索（HumanEval pass@1 曾達 92.7%）。要點是「失敗不只是回溯，還要記取教訓」。
