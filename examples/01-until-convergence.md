# until convergence（迭代到收斂）

- **族別**：一、迴圈類
- **控制骨架**：`while diff(本輪, 上一輪) 不為空: 再改一輪`
- **停止條件**：這一輪的輸出 ≈ 上一輪（差異為空 / 幾乎不動）

## Prompt 範例

```text
你要把下面這段文字改到「穩定」為止。規則：

1. 每一輪輸出兩個區塊：
   [DRAFT vN]  ── 這一輪的完整版本
   [DIFF]      ── 相對上一輪，你「實際改了什麼」的條列（每條 ≤1 句）

2. 若 [DIFF] 只剩下「無實質變更」或都是等價換字，就輸出：
   [CONVERGED] 並停止，不要再硬改。

3. 不准為了顯得有在做事而製造無意義的改動。

初始文字：
<<<
{{PASTE_TEXT}}
>>>
開始第 1 輪。
```

## 為什麼這樣做可以

- **把「停止」變成模型自己能判斷的訊號**。收斂迴圈最容易失敗的地方是模型「為了交差」每輪都亂改，永遠不收斂。要求它每輪輸出 `[DIFF]` 等於逼它把「這輪到底改了什麼」外顯出來——當 diff 空了，收斂條件就客觀可見，而不是靠感覺喊停。
- **禁止無意義改動**這條規則直接打掉 LLM 的「討好式修改」傾向：不加限制時，模型傾向持續換同義詞讓你以為它在進步。
- 適合「文字潤飾、把 code 改到穩定」這種**沒有硬性外部驗證、但有明確『再改也沒差』狀態**的任務。

## 什麼時候用 / 不要用

- ✅ 潤稿、rename、格式統一、把一段 code 重構到讀起來順。
- ❌ 有客觀對錯的任務（會不會編譯、測試綠不綠）——那種請直接升級成 [until oracle passes](05-until-oracle-passes.md)。

## 最大陷阱（來自來源文件的關鍵洞察）

> **收斂 ≠ 正確**。模型可能「穩定地停在一個錯誤答案上」。

收斂只證明「不再變動」，不證明「對」。只要你能弄到任何外部 ground truth（測試、編譯、lint、檢索），幾乎都該把停止條件換成 oracle。純內省式的收斂，在沒有外部訊號時甚至會**越改越差**（degeneration-of-thought）。
